# 实验报告模板

`report.md` 使用以下结构：

```markdown
# 受控实验：<主题>

- **实验日期**：YYYY-MM-DD
- **实验假设**：用户想验证的假设（如"暖光比冷光更有高级感"）
- **变量维度**：对比的维度（如"光线色温"）
- **变量值**：具体的对比组（如"暖光 3200K / 冷光 6500K"）
- **测试主体**：用什么场景/物体测试（如"哑光黑色无线音箱"）
- **生图模式**：api / codex

## 实验矩阵

实验矩阵摘要（引用 matrix.json）：
- 实验类型：单变量 A/B 测试 / 多风格快速对比
- base 描述：共享的固定部分
- runs 数量：N 组
- 每组变量值：列出各组名称

## 评分结果

引用 evaluation.json，以表格形式展示：

| Run | 主体 | 构图 | 遵循 | 材质 | 文字 | 稳定性 | 商业 | 总分 | 满分 |
|-----|------|------|------|------|------|--------|------|------|------|
| axis-lighting-warm | 4 | 3 | 4 | 4 | - | - | 4 | 19 | 25 |
| axis-lighting-cool | 4 | 3 | 5 | 3 | - | - | 2 | 17 | 25 |

注：`-` 表示该维度未评分（null）。

## 对比分析

- **总分排名**：最高分 → 最低分
- **维度级对比**：逐维度比较差异
- **关键差异**：差异最大的维度（说明该变量对效果影响最大）
- **胜出组合**：综合得分最高的变量值

## 结论

假设是否成立：✅ 成立 / ❌ 不成立 / ⚠️ 部分成立

具体说明：
- 哪些维度支持假设
- 哪些维度不支持
- 整体趋势

## 改进建议

如有进一步实验方向，列出建议：
- 可以追加哪些变量维度
- 当前实验的局限性
```

## 引用规范

- 矩阵引用：`matrix.json`
- 评分引用：`evaluation.json`
- 所有引用必须对应实际文件，不允许虚构
