# 研究报告模板

report.md 使用以下结构：

```markdown
# 研究：<主题>

- **研究日期**：YYYY-MM-DD
- **研究方向**：用户选择的研究主题
- **选择理由**：为什么选择这个方向（用户意图、覆盖缺口、素材可用性等）
- **涉及信息源**：本次研究使用了哪些信息源

## 来源

使用了哪些信息源，每个信息源收集了多少条素材（按 guide / prompt 分类统计）。

引用素材时使用 `[序号]` 标注，如 `[01]`、`[05-08]`。所有引用必须对应 `collections/` 中的实际文件。

## 发现

分析过程中观察到的关键发现。引用素材时标注来源，如"根据 [01] 的指导，..."。

## 提示词结构拆解

对 prompt 类型素材，按以下维度记录。每条分析标注素材编号：

- 主体清晰度
- 构图策略
- 风格锚点
- 光线描述
- 材质细节密度
- 色彩策略
- 文字规则
- 失败规避

## 提取的 pattern

从分析和指南中提炼的可复用模式。标注 pattern 的来源素材，如"综合 [02][07][12] 的共同特征"。

## 验证结果

生图验证的评分结果。引用 evaluations/ 下各轮评分文件。

## 结论

哪些确认有效，哪些需要进一步验证。
```

## 引用规范

- 素材引用格式：`[序号]`，对应 `collections/<序号>_<名称>.md`
- 批量引用：`[05-08]` 表示第 5 到 8 条素材
- 多条引用：`[02, 07, 12]` 表示第 2、7、12 条素材
- 评分引用：`evaluations/round-1.json`
- 所有引用必须对应实际文件，不允许虚构
