# 收集素材记录模板

保存路径：`research/studies/<date>-<topic>/collections/`

文件名格式：`<序号>_<来源简述>.md`，例如 `01_openai-academy.md`、`02_collected-mirror-selfie.md`

序号从 01 递增。

---

## 素材类型

| 类型 | 说明 | 来源示例 |
|------|------|----------|
| `guide` | 指南/文档类，提炼原则和技巧 | 官方文档、博客指南、教程 |
| `prompt` | Prompt 样例类，完整保留原文 | 社区画廊、collected/、推文 |

## 模板：guide 类型

```markdown
# <序号> <一句话简述>

- **来源**: [来源名称](URL 或原始文件路径)
- **来源类型**: 官方文档 / 博客指南 / 教程
- **可信度**: 一手来源 / 二手来源 / 社区来源
- **素材类型**: guide
- **采集时间**: YYYY-MM-DD

## 指导要点

<从原文提炼的关键原则、技巧、最佳实践，用项目符号列出>

## 原文摘要

<原文核心内容的精简摘录，保留关键表述>
```

## 模板：prompt 类型

```markdown
# <序号> <一句话简述>

- **来源**: [来源名称](URL 或原始文件路径)
- **来源类型**: 社区画廊 / 一手样本 / 博客示例
- **可信度**: 一手来源 / 二手来源 / 社区来源
- **素材类型**: prompt
- **采集时间**: YYYY-MM-DD

## Prompt

<prompt 内容，完整保留原文>

## 图片样例

<如有图片，记录 URL 或文件路径；无则删除此节>

## 备注

<研究相关的观察、初步分析、值得注意的特征>
```

## 规则

- 一个素材一个文件，按素材类型使用对应模板
- 即使素材来自本地目录（如 `collected/`），也要将所需内容**拷贝**到 `collections/`，不引用原始路径，不移动原文件
- prompt 类型的 Prompt 内容保留原文，不做改写
- guide 类型的指导要点是提炼而非照搬，保留关键表述但用自己的话总结
- 每个文件顶部元信息中 `素材类型` 字段必须标注，后续分析按此字段分流
