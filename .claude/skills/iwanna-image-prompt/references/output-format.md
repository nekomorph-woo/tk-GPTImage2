# 输出格式规范

## 最小模式（"直接生成 Prompt"入口）

````
你的提示词：

\`\`\`
创建一张[格式]，主体是[主体]。
...
\`\`\`

📐 流派：[流派名称]  📚 参考：#001, #034
````

## 教学模式（"聊聊我的想法" / "给我点灵感"入口）

### Part 1: 提示词

````
你的提示词：

\`\`\`
创建一张[格式]，主体是[主体]。
构图：...
视觉语言：...
光线：...
材质与细节：...
色彩：...
文字：...
质量约束：...
\`\`\`
````

## Part 2: 元信息

单行标注匹配流派、参考素材和 recipe：

```
📐 流派：[流派名称]  📚 参考：#001, #034, #042  📋 Recipe：[名称]（如适用）
```

## Part 3: Prompt 构造解析（教学卡）

每次输出附带一张 ASCII 教学卡，帮助用户理解"为什么这样写"。

### 选取规则

- 从 `genre-matching.md` 中当前匹配流派的 `教学要点` 选取
- 从质量预检中实际修正的问题中选取（如果有修正）
- 每次只选 1-2 个要点，不堆砌
- 优先选用户尚未见过的知识点（参考上下文判断）

### 知识来源

- 流派选择逻辑（genre-matching.md 每个流派的 `教学要点`）
- GRIMOIRE 第 4 节关键心法（7 条原则）
- GRIMOIRE 第 8 节失败诊断（8 种模式）
- 质量预检中实际触发并修正的问题

### 模板

````
┌─── Prompt 构造解析 ───────────────────────────┐
│                                                │
│  🎯 为什么选 [流派名称]                         │
│  ├─ [流派选择原因，1-2 句]                      │
│                                                │
│  🔑 关键心法                                    │
│  ├─ [本次应用的核心原则，引用 GRIMOIRE 心法]    │
│                                                │
│  ⚠️ 规避的坑                                   │
│  ├─ [本次构造中主动规避的常见失败模式]          │
│                                                │
└────────────────────────────────────────────────┘
````

### 示例

````
你的提示词：

\`\`\`
创建一张高级产品海报：一个半透明无线音箱放在磨砂亚克力底座上...
\`\`\`

📐 流派：高级产品海报  📚 参考：#021, #048

┌─── Prompt 构造解析 ───────────────────────────┐
│                                                │
│  🎯 为什么选 高级产品海报                       │
│  ├─ 产品主视觉需要"单一主体+受控反射+留白"，    │
│  │  这是让产品脱颖而出的核心结构。              │
│                                                │
│  🔑 关键心法                                    │
│  ├─ "材质要具体" — 写"磨砂亚克力"比写"好看底座"│
│  │  有效得多，模型需要具体的表面描述才能还原。  │
│                                                │
│  ⚠️ 规避的坑                                   │
│  ├─ 没有堆叠"masterpiece, ultra detailed"——    │
│  │  模糊质量词会让图看起来像 AI 生成。          │
│                                                │
└────────────────────────────────────────────────┘
````
