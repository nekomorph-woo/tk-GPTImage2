# 流派匹配规则

## Part 1: 核心公式

### 提示词骨架

```text
创建一张[格式]，主体是[主体]。
目的：[这张图要用于什么]。
构图：[镜头角度、裁切、主体位置、前景/中景/背景、留白]。
视觉语言：[一个清晰风格锚点]。
光线：[方向、软硬、色温、对比度]。
材质与细节：[表面、纹理、镜头、印刷、织物、皮肤、金属、玻璃]。
色彩：[2-4 个主色和一个强调色]。
文字：[引号中的精确文字、位置、字体规则，或"无文字"]。
质量约束：[具体需要避免的问题]。
```

### 规则

1. 不要堆叠"masterpiece, ultra detailed"等模糊质量词
2. 中文海报：精确中文标题放引号内，声明"不要额外随机文字"
3. 质量约束写具体失败场景（"手部连贯""不要拼写错误"），不写空泛赞美词

### 中英混合原则

创意构思可以先用中文完成，最终生成提示词可以使用精确英文，也可以使用结构清晰的中文。专有名词、精确标题和必须出现的文字保持原样。

## Part 2: 八大流派

### 5.1 写实摄影

- **适用场景**：生活感照片、街拍、iPhone/raw 质感、室内外真实场景
- **特征词**：照片、自拍、街拍、生活、日常、iPhone、raw、纪实、随拍、portrait、photo
- **公式**：真实场景 + 相机来源 + 时间/地点 + 普通生活细节 + 光源 + 轻微瑕疵
- **负面约束**：通用、技术质量
- **教学要点**：
  - 写实感的核心不是"像照片"三个字，而是**具体的时间和地点**——"晚上 10 点便利店门口"比"一张街头照片"有效 10 倍
  - 声明"允许轻微运动模糊"和"反射真实"这类"不完美"反而增强真实感，完美无瑕的照片看起来更像 AI
  - 心法：**光线要像真实拍摄一样写**——从哪里来、软硬、冷暖，而不是写"好看的光"

```text
创建一张完全未处理的 iPhone 照片：晚上 10 点，雨中的便利店门口。三位普通人在玻璃门旁短暂聊天，一人拿着饮料，一人看手机，一人坐在塑料椅上。明亮白色店内光从玻璃透出，暖色路灯反射在湿润路面上，背景能看到共享单车和小海报。行人面部和姿态自然，不要网红摆拍感，允许轻微运动模糊，反射真实，无叠加文字，无水印。
```

### 5.2 电影剧照

- **适用场景**：氛围大片、故事感场景、封面视觉
- **特征词**：电影、剧照、氛围、故事、大片、cinematic、film、moody、dramatic
- **公式**：角色 + 精确地点 + 镜头/裁切 + 真实光源 + 氛围物件 + 克制调色
- **负面约束**：通用、人物、光影与色彩
- **教学要点**：
  - 电影感的秘诀是**克制**——只选 2-4 个主色（如"钢灰+褪色青+小面积红"），颜色越多越廉价
  - "氛围物件"（售货机、长椅、雾气）不是装饰，它们定义了**空间的故事感**，让画面从"好看"升级为"有叙事"
  - 心法：**构图写在细节前**——模型先理解画面骨架（中远景 35mm、主体偏离中心），再填充材质和光线

```text
创建一张真实克制的电影剧照：一名快递员独自站在深夜空荡的火车站里。中远景 35mm 构图，主体略微偏离中心，长长的站台线条向背景延伸。天花板是冷色荧光灯，右侧有一台暖光自动售货机，空气中有轻微雾气。湿润外套面料、磨旧运动鞋、金属长椅、瓷砖反光。调色：钢灰、褪色青色，售票机带来小面积红色强调。无文字，无水印，手部和面部自然。
```

### 5.3 高级产品海报

- **适用场景**：产品主视觉、电商头图、社媒新品图
- **特征词**：产品、海报、海报广告、电商、主视觉、product、poster、ad、commercial
- **公式**：单一产品 + 表面/底座 + 受控反射 + 留白 + 短标题 + Logo 限制
- **负面约束**：产品、字体排版、技术质量、光影与色彩
- **教学要点**：
  - "留白"是产品海报最重要的构图决策——它让产品成为绝对视觉焦点，信息层级只有一层
  - 心法：**材质要具体**——"磨砂亚克力"比"好看的底座"有效得多，模型需要具体的表面描述才能还原真实质感
  - "不要虚假品牌 Logo"不是建议而是必须——模型会随机生成不存在的品牌标识，污染商业用途

```text
创建一张高级产品海报：一个半透明无线音箱放在磨砂亚克力底座上。产品居中作为英雄主体，低机位，上方有大量留白。左上方大型柔光箱反射，边缘有细窄轮廓光，接触阴影干净。材质：半透明外壳、拉丝铝网罩、微妙内部光，不要指纹。调色：清透玻璃、石墨灰、冷白，一个小面积青柠色强调。文字：顶部出现"SOUND, SEEN"，锐利现代无衬线字体，不要其他文字，不要虚假品牌 Logo。
```

### 5.4 中文字体海报

- **适用场景**：中文封面、活动主视觉、诗意海报
- **特征词**：中文海报、封面、标题、字体、诗意、chinese、typography、editorial
- **公式**：短中文标题 + 标题区域 + 主体区域 + 印刷/编辑质感 + 严格无额外文字规则
- **负面约束**：中文文字、字体排版
- **教学要点**：
  - 中文海报最大的坑是**文字失控**——模型会随机生成乱码中文字符，必须声明"不要额外随机中文字符，不要英文文字"
  - 心法：**文字越短越好**——只保留一句核心标题，放在引号内，说明位置和字体气质
  - "印刷杂志质感"这类风格锚点比"高级感"有效——后者是模糊形容词，前者是模型能理解的视觉语言

```text
创建一张竖版高级中文编辑海报，标题为"风经过城市"。精确中文标题大而清晰地出现在上三分之一，使用优雅的现代宋体感字体，浅灰底上的黑色墨迹。画面下方中央是一位在湿润城市街道上撑着半透明雨伞的人。雨夜反光、店铺柔光、路面银色高光。印刷杂志质感，布局克制，不要额外随机中文字符，不要英文文字，不要水印。
```

### 5.5 角色一致性

- **适用场景**：IP 角色、连续漫画、头像、故事设定
- **特征词**：角色、IP、一致性、连续、漫画、character、consistent、avatar、OC
- **公式**：两步法 — 先做角色参考图（全身/纯色背景/正面）→ 再用编辑放入场景
- **负面约束**：通用、人物、解剖细节
- **教学要点**：
  - 角色一致性是 GPT Image 2 最难的事情之一，纯文本**不可能**保持角色身份——必须用两步法：先生成参考图，再用编辑
  - 第一步参考图的关键是**纯净背景 + 全身 + 正面**——背景越简单，模型越能专注于角色特征
  - 心法：**要稳定角色，不要只靠文本反复生成；先做参考图，再用编辑保持身份和结构**
  - 编辑时要明确说"保留什么"和"不要改变什么"——这是编辑图三条咒律的核心

**第一步 — 角色参考图**：

```text
创建一张全身角色参考图：Lin，一位年轻城市探险者。背景为纯净暖灰色摄影棚，放松的三分之四站姿，脸和手清晰。Lin 有黑色短发和一个小银色发夹，表情平静而好奇，穿苔绿色野外夹克、奶油色围巾、帆布挎包、磨旧皮靴，带一个小黄铜指南针。精致动画电影概念设定，柔和棚拍光，服装细节连贯，不要额外配饰，不要文字。
```

**第二步 — 场景编辑**：

```text
使用参考角色，保留 Lin 的脸、发型、服装、挎包、靴子和黄铜指南针。把 Lin 放在黄昏时安静的图书馆档案室里，抬头看向暖色光束中的漂浮尘埃。保持身份和服装一致；只改变环境和姿势。
```

### 5.6 3D 图标与小物件

- **适用场景**：App icon、贴纸、UI 资产、品牌视觉元素
- **特征词**：3D、图标、icon、贴纸、sticker、等距、isometric、UI 资产
- **公式**：单一物体 + 材质组合 + 等距视角 + 棚拍光 + 干净边距 + 无文字
- **负面约束**：产品
- **教学要点**：
  - 3D 图标的"等距三分之四视角"是行业标准角度——它让物体看起来立体且可识别，正面或侧面都做不到
  - 心法：**一个 prompt 只放一个强风格锚点**——"半透明玻璃+金色金属框架+薄荷色叶片"是三种材质组合（这是具体描述），而非三种风格混合
  - "不要背景杂乱，不要字母，不要 Logo，不要重复"——3D 图标最忌讳画面中出现多余元素，干净边距是关键

```text
创建一个单体高级 3D App 图标：一座微型玻璃温室，里面有一株发光植物。等距三分之四视角，物体居中，边距干净，柔和棚拍光，地面有微妙阴影。材质：半透明玻璃面板、圆角浅金色金属框架、发光薄荷色叶片、抛光底座。只出现一个物体，不要背景杂乱，不要字母，不要 Logo，不要重复温室。
```

### 5.7 UI / 社媒图

- **适用场景**：封面、App 概念图、仪表盘视觉、帖子配图
- **特征词**：UI、社媒、封面、仪表盘、App、界面、mockup、social、dashboard、cover
- **公式**：屏幕用途 + 布局网格 + 真实内容层级 + 设备/环境 + 品牌色板 + 无占位符杂乱感
- **负面约束**：字体排版、风格控制
- **教学要点**：
  - UI 截图最容易出现"占位符杂乱"——模型会在屏幕上随机填满文字和图标，必须声明"不要虚假 App Logo，不要额外文字"
  - "品牌色板"是控制 UI 视觉一致性的关键——限定 3-4 个主色（如"瓷白+石墨灰+鼠尾草绿+珊瑚色"），让画面有设计系统感
  - 心法：**构图写在细节前**——先确定"宽幅 16:9，干净桌面上有平板"，再填充屏幕内容和环境光线

```text
创建一张精致的社媒封面图，用于生产力 App 发布。宽幅 16:9 构图，干净桌面上有一块真实平板屏幕，屏幕显示专注的周计划界面，包含卡片、日历块和进度环。编辑感产品摄影，柔和晨间窗光，旁边有纸质笔记本和铅笔。调色：瓷白、石墨灰、鼠尾草绿、小面积珊瑚色强调。文字：左上角出现大而可读的"PLAN THE WEEK"，不要额外文字，不要虚假 App Logo。
```

### 5.8 信息图与教学图

- **适用场景**：知识卡片、科普图、流程说明
- **特征词**：信息图、教学、知识卡片、科普、流程、infographic、educational、diagram
- **公式**：一个主题 + 3-5 个区块 + 清晰标签 + 图标风格 + 有限色板 + 可读性规则
- **负面约束**：字体排版
- **教学要点**：
  - 信息图最常见的失败是"细小不可读标签"——模型会在小区块里塞满密密麻麻的文字，必须声明"只使用大而可读的标签"
  - "3-5 个区块"是信息图的信息密度上限——超过 5 个区块画面会变得杂乱，信息层级崩塌
  - "图标一致"和"有限色板"是信息图看起来专业的两个关键——杂乱的图标风格和过多颜色会让它看起来像小学生的手抄报

```text
创建一张干净的教学信息图，用四个带标签步骤解释"雨是如何形成的"。竖版海报，四个纵向堆叠面板，用简单矢量风插图表现：蒸发、凝结、云形成、降雨。只使用大而可读的标签："蒸发""凝结""云""降雨"。安静科学教室风格，白色背景，蓝绿色色板，图标一致，不要额外文字，不要细小不可读标签。
```

## Part 3: 编辑图规则

编辑比从零生成更适合稳定控制。编辑 prompt 必须说清三件事：

- **保留什么**：身份、构图、产品形状、布局、文字位置
- **改变什么**：光线、背景、颜色、材质、局部元素
- **不要动什么**：脸、手、Logo、主体比例、边缘轮廓

模板：

```text
保留参考图中的[身份/构图/产品形状/布局]。
只改变[目标层]。
新的视觉方向：[风格、光线、材质、色彩]。
不要改变[受保护细节]。
移除/避免[失败模式]。
```

## Part 4: 失败诊断

| 症状 | 原因 | 修正 |
|------|------|------|
| 图很普通 | 缺少具体地点、真实物件、风格锚点 | 补充具体场景细节和真实物件 |
| 图很乱 | 主体、背景、文字都在抢焦点 | 减少对象数量，明确视觉层级 |
| 角色漂移 | 纯文本生成无法保持身份一致 | 先做角色参考图，再用编辑 |
| 产品变形 | 文本无法精确控制产品轮廓 | 用参考图，声明"保留准确轮廓和比例" |
| 中文乱码 | 中文字符过多或无约束 | 缩短中文只保留一句，声明"只出现精确文字，不要其他文字" |
| 像 AI 图 | 堆叠"超高清大师级"等模糊词 | 减少 AI 味堆词，增加真实拍摄缺陷、具体材质和光源 |
| 构图散 | 缺少明确的构图指令 | 明确主体位置、镜头角度和留白 |
| 质感廉价 | 颜色过多、缺少材质描述 | 减少颜色，写清反射、阴影、材质和接触面 |
