# Idea 文件格式

`ideas/` 工作空间的文件规范。

## 命名规则

```
<ISO日期>T<时间>-<slug>.md
```

例: `2026-05-04T14-30-00Z-midnight-noodle.md`

slug 取概念标题的英文简写（kebab-case），不超过 5 个单词。

## 文件结构

```markdown
# <概念标题>

## 源头  ⏺ 稳定

用户为什么想生这张图。一句话到一段话，记录需求的来龙来脉——工作任务、生活瞬间、情绪表达、某个想可视化的脑海画面等。仅当用户主动修改概念时更新。

## 概念描述  ⏺ 稳定

步骤 0B1 输出。用户确认后的"小作文"——用日常语言唤起画面感的场景描述（调动感官、具象替代抽象、制造临场感、日常经验类比、留细节但不填满），不含 prompt 技术术语。仅当用户主动修改概念时更新。

## 技术解读  ⏺ 稳定

步骤 0B2 输出。左右对照格式，展示概念如何翻译成流派/构图/光线/色彩/材质等技术维度。仅当概念变更导致技术维度变化时更新。

## Prompt 历史  📎 追加

### v1 — <日期>

（提示词文本）

修改原因：初始版本

每次 prompt 修改追加新版本，记录修改原因。不覆盖旧版本，保留完整审计轨迹。

### v2 — <日期>

（提示词文本）

修改原因：<用户要求的具体调整>

## 生成记录  📎 追加

可选。用户后续用 `/generate-image` 生图后追加，格式：

- `gen/outputs/<timestamp>-<name>/` — 备注（如"第一次尝试"、"调整了光线"）
```

### 更新策略

| 策略 | 字段 | 说明 |
|------|------|------|
| ⏺ 稳定 | 源头、概念描述、技术解读 | 仅当用户主动修改概念时更新；仅调 prompt 时不动 |
| 📎 追加 | Prompt 历史、生成记录 | 只追加新条目，不覆盖旧内容 |

## 会话续接

用户引用 `ideas/` 中已有文件时：
1. 读取该文件，跳过步骤 0（概念已存在）
2. 根据用户指令决定从哪个阶段继续（修改概念 / 重新匹配流派 / 直接用已有 prompt 生图）
3. 完成后更新同一文件，不创建新文件
