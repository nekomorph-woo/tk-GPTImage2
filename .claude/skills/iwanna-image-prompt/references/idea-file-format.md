# Idea 文件格式

`ideas/` 工作空间的文件规范。

## 命名规则

```
<ISO日期>T<时间>-<slug>.md
```

例: `2026-05-04T14-30-00Z-midnight-noodle.md`

slug 取概念标题的英文简写（kebab-case），不超过 5 个单词。

## 文件结构

```markdown
# <概念标题>

## 源头

用户为什么想生这张图。一句话到一段话，记录需求的来龙去脉——工作任务、生活瞬间、情绪表达、某个想可视化的脑海画面等。

## 概念描述

步骤 0B1 输出。用户确认后的"小作文"——用日常语言唤起画面感的场景描述（调动感官、具象替代抽象、制造临场感、日常经验类比、留细节但不填满），不含 prompt 技术术语。

## 技术解读

步骤 0B2 输出。左右对照格式，展示概念如何翻译成流派/构图/光线/色彩/材质等技术维度。

## Prompt 历史

### v1 — <日期>

（提示词文本）

修改原因：初始版本

续接修改时追加新版本，记录修改原因。不覆盖旧版本，保留完整审计轨迹。

### v2 — <日期>

（提示词文本）

修改原因：<用户要求的具体调整>

## 生成记录

可选。用户后续用 `/generate-image` 生图后追加，格式：

- `gen/outputs/<timestamp>-<name>/` — 备注（如"第一次尝试"、"调整了光线"）
```

## 会话续接

用户引用 `ideas/` 中已有文件时：
1. 读取该文件，跳过步骤 0（概念已存在）
2. 根据用户指令决定从哪个阶段继续（修改概念 / 重新匹配流派 / 直接用已有 prompt 生图）
3. 完成后更新同一文件，不创建新文件
