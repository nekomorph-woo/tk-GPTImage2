# Prompt 内容判断规则

步骤 4 语义判断时使用。阅读提取到的推文/评论内容后，匹配以下规则决定后续操作。

> **发现新规则时**，立即将本文件追加新规则条目，设置创建者为 `自动发现`。

---

## 规则列表

### R1: 正文包含完整 prompt

- **条件**: 主推文正文包含可直接用于生图的 prompt 文本（具体描述了场景、风格、参数等）
- **操作**: 提取正文中的 prompt，跳过评论检查，进入步骤 5
- **创建时间**: 2026-05-02
- **创建者**: 自动创建
- **示例**: 正文为 JSON 格式 prompt、或详细的场景描述文本

### R2: 引导文案，prompt 在评论区

- **条件**: 主推文正文是简短引导（如 "Prompt below 👇"、"See reply"、"prompts below"），不包含实际 prompt 内容
- **操作**: 检查提取到的博主评论（author_comments），找到包含 prompt 的评论作为 prompt 来源
- **创建时间**: 2026-05-02
- **创建者**: 自动创建

### R3: 外部链接，prompt 不在本页

- **条件**: 正文或评论中的 prompt 指向外部链接（如 promptsref.com、其他平台链接），而非直接包含文本
- **操作**: 跳过该条推文，报告用户跳过原因
- **创建时间**: 2026-05-02
- **创建者**: 自动创建
- **示例**: "prompt：https://promptsref.com/tool/..."

### R4: 非 prompt 内容

- **条件**: 正文是简单的感慨、评价、随想、社交互动，不包含任何 prompt 或引导信息
- **操作**: 跳过该条推文，报告用户跳过原因
- **创建时间**: 2026-05-02
- **创建者**: 自动创建
- **示例**: "This year's student teachers are way too hot, right...?"、"GM, I reran my previous JSON prompt..."（纯叙述无 prompt 内容）

---

## 判断流程

```
提取主推文正文
  ├─ 匹配 R1 → 提取 prompt，进入步骤 5
  ├─ 匹配 R2 → 检查博主评论，找到 prompt 进入步骤 5
  ├─ 匹配 R3 → 报告用户：外部链接，无文本 prompt
  ├─ 匹配 R4 → 报告用户：非 prompt 内容
  └─ 均不匹配 → 视为 R4 处理
```

如果 R2 路径下博主评论都是社交回复，未找到 prompt：
- 视为 R4 处理，报告用户
- 若该情况频繁出现，考虑新增规则并记录到此文件
