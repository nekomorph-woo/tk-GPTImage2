# GPT Image2 提示词魔导书

版本：0.1  
日期：2026-05-01  
主战场：`tk_GPTImage2`

这本手册的目标不是收集一堆漂亮咒语，而是建立一套能反复出好图的工作法：观察、拆解、成型、批量实验、复盘、沉淀。

## 1. 先确立模型事实

当前工作台默认使用 `gpt-image-2`。根据 OpenAI 官方文档，它是当前 GPT Image 系列的图片生成与编辑模型，支持文本输入、图片输入和图片输出，可用于 `v1/images/generations` 与 `v1/images/edits`。官方图片生成指南也确认：只做单次生成或单次编辑时，Image API 最直接；如果要做多轮对话式编辑，则 Responses API 更合适。

实战参数：

- `model`: `gpt-image-2`
- `size`: 常用 `1024x1024`、`1536x1024`、`1024x1536`，也可用更大尺寸；边长需符合官方约束。
- `quality`: `low` 用于草稿，`medium` 用于常规实验，`high` 用于最终交付。
- `n`: 同一 prompt 一次生成多张，适合小批量比较。
- `output_format`: `png` 默认稳，`jpeg` 更快，`webp` 适合网页。
- `background`: `gpt-image-2` 当前不支持透明背景，别把透明图当成稳定路径。

## 2. 魔导书的核心公式

高质量 prompt 通常不是更长，而是更清楚。官方建议把提示词落到目的、主体、发生什么、地点、风格；如果构图、光线、约束重要，也要写清楚。我们的增强版公式是：

```text
目的 + 主体 + 构图 + 风格锚点 + 光线 + 材质细节 + 色彩策略 + 文字规则 + 失败规避 + 迭代指令
```

每次只要这十个位置清楚，图像会明显更稳定。

## 3. 万用提示词骨架

```text
创建一张[格式]，主体是[主体]。
目的：[这张图要用于什么]。
构图：[镜头角度、裁切、主体位置、前景/中景/背景、留白]。
视觉语言：[一个清晰风格锚点]。
光线：[方向、软硬、色温、对比度]。
材质与细节：[表面、纹理、镜头、印刷、织物、皮肤、金属、玻璃]。
色彩：[2-4 个主色和一个强调色]。
文字：[引号中的精确文字、位置、字体规则，或“无文字”]。
质量约束：[具体需要避免的问题]。
```

中文构思可以更自由，最终生成提示词也可以使用结构清晰的中文。中文文字海报要保留精确中文标题，并加上“不要额外乱码文字”。

## 4. 关键心法

- 一个提示词只放一个强风格锚点。比如“编辑感产品摄影”比“电影感、动漫、超写实、赛博朋克、杂志、3D”更稳。
- 构图写在细节前。模型先理解画面骨架，再填充材质。
- 光线要像真实拍摄一样写：从哪里来、软硬、冷暖、阴影怎么落。
- 材质要具体：哑光涂层、拉丝金属、湿润路面、瓷感皮肤纹理、Risograph 纸张颗粒。
- 文字越短越好。把文字放进引号，说明位置、字体感觉、字号关系，并声明不要出现其他文字。
- 要稳定角色或产品，不要只靠文本反复生成；先做参考图，再用编辑保持身份和结构。
- 迭代时一次只改一个变量：光线、构图、色彩、材质、文字，不要一起大改。

## 5. 八大流派

### 5.1 写实摄影

适合：生活感照片、街拍、iPhone/raw 质感、室内外真实场景。

公式：

```text
真实场景 + 相机来源 + 时间/地点 + 普通生活细节 + 光源 + 轻微瑕疵
```

咒语：

```text
创建一张完全未处理的 iPhone 照片：晚上 10 点，雨中的便利店门口。三位普通人在玻璃门旁短暂聊天，一人拿着饮料，一人看手机，一人坐在塑料椅上。明亮白色店内光从玻璃透出，暖色路灯反射在湿润路面上，背景能看到共享单车和小海报。行人面部和姿态自然，不要网红摆拍感，允许轻微运动模糊，反射真实，无叠加文字，无水印。
```

### 5.2 电影剧照

适合：氛围大片、故事感场景、封面视觉。

公式：

```text
角色 + 精确地点 + 镜头/裁切 + 真实光源 + 氛围物件 + 克制调色
```

咒语：

```text
创建一张真实克制的电影剧照：一名快递员独自站在深夜空荡的火车站里。中远景 35mm 构图，主体略微偏离中心，长长的站台线条向背景延伸。天花板是冷色荧光灯，右侧有一台暖光自动售货机，空气中有轻微雾气。湿润外套面料、磨旧运动鞋、金属长椅、瓷砖反光。调色：钢灰、褪色青色，售票机带来小面积红色强调。无文字，无水印，手部和面部自然。
```

### 5.3 高级产品海报

适合：产品主视觉、电商头图、社媒新品图。

公式：

```text
单一产品 + 表面/底座 + 受控反射 + 留白 + 短标题 + Logo 限制
```

咒语：

```text
创建一张高级产品海报：一个半透明无线音箱放在磨砂亚克力底座上。产品居中作为英雄主体，低机位，上方有大量留白。左上方大型柔光箱反射，边缘有细窄轮廓光，接触阴影干净。材质：半透明外壳、拉丝铝网罩、微妙内部光，不要指纹。调色：清透玻璃、石墨灰、冷白，一个小面积青柠色强调。文字：顶部出现“SOUND, SEEN”，锐利现代无衬线字体，不要其他文字，不要虚假品牌 Logo。
```

### 5.4 中文字体海报

适合：中文封面、活动主视觉、诗意海报。

公式：

```text
短中文标题 + 标题区域 + 主体区域 + 印刷/编辑质感 + 严格无额外文字规则
```

咒语：

```text
创建一张竖版高级中文编辑海报，标题为“风经过城市”。精确中文标题大而清晰地出现在上三分之一，使用优雅的现代宋体感字体，浅灰底上的黑色墨迹。画面下方中央是一位在湿润城市街道上撑着半透明雨伞的人。雨夜反光、店铺柔光、路面银色高光。印刷杂志质感，布局克制，不要额外随机中文字符，不要英文文字，不要水印。
```

### 5.5 角色一致性

适合：IP 角色、连续漫画、头像、故事设定。

第一步先做角色参考图：

```text
创建一张全身角色参考图：Lin，一位年轻城市探险者。背景为纯净暖灰色摄影棚，放松的三分之四站姿，脸和手清晰。Lin 有黑色短发和一个小银色发夹，表情平静而好奇，穿苔绿色野外夹克、奶油色围巾、帆布挎包、磨旧皮靴，带一个小黄铜指南针。精致动画电影概念设定，柔和棚拍光，服装细节连贯，不要额外配饰，不要文字。
```

第二步再做场景编辑：

```text
使用参考角色，保留 Lin 的脸、发型、服装、挎包、靴子和黄铜指南针。把 Lin 放在黄昏时安静的图书馆档案室里，抬头看向暖色光束中的漂浮尘埃。保持身份和服装一致；只改变环境和姿势。
```

### 5.6 3D 图标与小物件

适合：App icon、贴纸、UI 资产、品牌视觉元素。

公式：

```text
单一物体 + 材质组合 + 等距视角 + 棚拍光 + 干净边距 + 无文字
```

咒语：

```text
创建一个单体高级 3D App 图标：一座微型玻璃温室，里面有一株发光植物。等距三分之四视角，物体居中，边距干净，柔和棚拍光，地面有微妙阴影。材质：半透明玻璃面板、圆角浅金色金属框架、发光薄荷色叶片、抛光底座。只出现一个物体，不要背景杂乱，不要字母，不要 Logo，不要重复温室。
```

### 5.7 UI / 社媒图

适合：封面、App 概念图、仪表盘视觉、帖子配图。

公式：

```text
屏幕用途 + 布局网格 + 真实内容层级 + 设备/环境 + 品牌色板 + 无占位符杂乱感
```

咒语：

```text
创建一张精致的社媒封面图，用于生产力 App 发布。宽幅 16:9 构图，干净桌面上有一块真实平板屏幕，屏幕显示专注的周计划界面，包含卡片、日历块和进度环。编辑感产品摄影，柔和晨间窗光，旁边有纸质笔记本和铅笔。调色：瓷白、石墨灰、鼠尾草绿、小面积珊瑚色强调。文字：左上角出现大而可读的“PLAN THE WEEK”，不要额外文字，不要虚假 App Logo。
```

### 5.8 信息图与教学图

适合：知识卡片、科普图、流程说明。

公式：

```text
一个主题 + 3-5 个区块 + 清晰标签 + 图标风格 + 有限色板 + 可读性规则
```

咒语：

```text
创建一张干净的教学信息图，用四个带标签步骤解释“雨是如何形成的”。竖版海报，四个纵向堆叠面板，用简单矢量风插图表现：蒸发、凝结、云形成、降雨。只使用大而可读的标签：“蒸发”“凝结”“云”“降雨”。安静科学教室风格，白色背景，蓝绿色色板，图标一致，不要额外文字，不要细小不可读标签。
```

## 6. 编辑图的三条咒律

编辑比从零生成更适合稳定控制。写编辑 prompt 时必须说清楚三件事：

- 保留什么：身份、构图、产品形状、布局、文字位置。
- 改变什么：光线、背景、颜色、材质、局部元素。
- 不要动什么：脸、手、Logo、主体比例、边缘轮廓。

模板：

```text
保留参考图中的[身份/构图/产品形状/布局]。
只改变[目标层]。
新的视觉方向：[风格、光线、材质、色彩]。
不要改变[受保护细节]。
移除/避免[失败模式]。
```

## 7. 批量实验法

每次实验不要凭感觉乱试。用三层矩阵：

- 固定层：主体、画幅、目标用途。
- 变量层：风格、光线、构图、色彩，只改一个。
- 评估层：主体清晰度、构图、文字、材质、商业可用性、是否可复现。

建议流程：

1. `quality=low` 跑 6-12 张探索方向。
2. 选 2 张最有潜力的提示词。
3. 把构图和主体锁死，只改光线或色彩。
4. 最终图用 `quality=high`。
5. 把成功提示词写回 `studies/`。

## 8. 失败诊断

- 图很普通：缺少具体地点、真实物件、风格锚点。
- 图很乱：主体、背景、文字都在抢焦点；减少对象数量。
- 角色漂移：先做角色参考图，再用编辑。
- 产品变形：用参考图，声明“保留准确轮廓和比例”。
- 中文乱码：缩短中文，只保留一句，声明“只出现精确文字，不要其他文字”。
- 像 AI 图：减少“超高清大师级”堆词，增加真实拍摄缺陷、具体材质和光源。
- 构图散：明确主体位置、镜头角度和留白。
- 质感廉价：减少颜色，写清反射、阴影、材质和接触面。

## 9. 我自己的研究流程

每轮探索按这个顺序做：

1. 搜索公开案例和官方资料。
2. 只提炼结构，不迷信原提示词。
3. 把案例归入八大流派。
4. 写一个干净公式。
5. 生成 3-5 个变体。
6. 记录哪个变量带来提升。
7. 把稳定套路加入 `recipes/`。

## 10. 核心判断

GPT Image2 的提示词工程，本质不是“神秘咒语”，而是视觉导演能力。你要告诉模型：

- 这张图为什么存在。
- 谁是主角。
- 镜头在哪里。
- 光从哪里来。
- 什么材质应该被看见。
- 哪些东西绝对不要出现。
- 下一轮只改哪一层。

当这些信息稳定，出图就会稳定。
