# 研究：角色设定集与多视角生成

- **研究日期**：2026-05-04
- **研究方向**：通过先生成角色设定集（character reference sheet）建立多视角 baseline，再基于设定集生成新场景
- **选择理由**：角色设定集是建立一致性的重要中间产物，适合需要大量同角色产出的场景。collected/ 中 #076 是 production bible 模板，#018 是 character design chart。
- **涉及信息源**：Dreamina、SelfieLab、BigPromptHub、Pixverse、NemoVideo、GLBGPT、项目内部魔导书

## 来源

### 来源：Dreamina（[01]）

根据 [01] 的指导：

**核心原则**
- 身份锚点（Identity Anchors）是角色一致性的基础——在 prompt 中固定角色的发色、瞳色、服装、体型等描述词，确保多次生成之间角色不变（来源 [01]）
- 3:2 横版比例是转角度表的最佳画幅，提供足够横向空间展示多角度视图（来源 [01]）
- T-pose 适合角色设计（展示轮廓），A-pose 适合品牌吉祥物（自然站姿）（来源 [01]）

**技巧模式**
- 使用 "consistent proportions"、"exact same face/body proportions"、"identical styling" 等一致性关键词强化跨面板一致性（来源 [01]）
- 表情面板可在转角度表中一并生成，将表情名称作为附加面板指令（来源 [01]）
- 纯色背景（纯白影棚背景）便于后期抠图（来源 [01]）
- 5 步工作流：定义角色 → 编写整合 prompt → 配置布局 → 生成 → 细化导出（来源 [01]）

**反面案例**
- 未指定固定角色描述词会导致多面板间角色外观漂移（来源 [01]）
- 纯色背景不统一会影响后期抠图效率（来源 [01]）

**可操作项**
- 直接复用 [01] 中的 4 套模板结构（男主、女盗贼、角色包、吉祥物），替换角色描述即可生成新角色设定集

### 来源：SelfieLab（[02]）

根据 [02] 的指导：

**核心原则**
- 参考图锚定模式是角色一致性的核心方法：先生成一张干净的纯色背景角色参考图，后续所有生成都引用该图作为身份锚（来源 [02][07]）
- 角色描述词必须逐字复用，不可使用近义词替换（来源 [02][05]）
- 从简单布局开始，确认角色一致后再增加复杂度（来源 [02]）

**技巧模式**
- 核心公式：`[Reference Image] + [Pose/Scene Description] + [Style/Details] + [Consistency Keywords]`（来源 [02]）
- 一致性关键词放在 prompt 尾部作为"不变量指令"（来源 [02]）
- 使用 edit 端点（非 generate 端点），支持最多 16 张参考图输入（来源 [02]）
- 3x3 Grid 布局是表情/姿势表的经典布局，每个格子明确指定姿势和表情名称（来源 [02]）
- "Same Character" 模式：在 prompt 中写 "same character, do not change"，配合参考图实现跨场景一致性（来源 [02]）
- 一致性关键词清单："exact same face"、"identical build"、"same skin tone"、"same proportions"、"do not change appearance"、"maintain identity throughout"（来源 [02]）

**反面案例**
- 使用近义词替换角色描述词会导致一致性下降（来源 [02]）
- 不使用参考图直接生成多面板角色表，一致性难以保证（来源 [02][07]）

**可操作项**
- "Using this reference image exactly for the character's face, build, and skin tone" 可直接作为 edit prompt 的核心句式（来源 [02]）
- 3x3 网格结构可直接用于碧蓝航线角色的表情/姿势展示表

### 来源：BigPromptHub（[03]）

根据 [03] 的指导：

**核心原则**
- Reference-Interference 技术：双图引用各司其职——img1 承担语义锚（角色身份），img2 承担结构约束（转角度格式）（来源 [03]）

**技巧模式**
- 提取 Prompt 核心句式："extract the [character] from [img1] using the character turnaround format from [img2]"（来源 [03]）
- 适用于已有角色单图、需要快速生成完整转角度表的场景（来源 [03]）

**反面案例**
- 角色细节可能不完美保留，建议生成后用额外 prompt 细化（来源 [03]）
- 不适合需要极高精度还原的角色设计（来源 [03]）

**可操作项**
- 对碧蓝航线已有角色立绘，可直接用 Reference-Interference 技术提取转角度表，省去手动描述步骤

## 发现

### 1. 角色设定集的核心价值

角色设定集（character reference sheet / turnaround / model sheet）是达成高一致性角色生成的关键中间产物。所有来源一致认同：**先生成完整的角色设定集，再将设定集作为参考图进行后续场景生成**，是比每次从文本描述重新生成更可靠的一致性方案（来源 [01][02][07]）。

### 2. 两种互补的工作流

来源 [02] 和 [07] 描述了本质相同的两步工作流：
- **Step 1**：在干净的纯色背景上生成角色参考图（建立身份锚点）
- **Step 2**：使用 edit 端点引用参考图，在 prompt 中明确"保留身份，只改场景/姿势"

来源 [03] 补充了第三条路径——**Reference-Interference**：当已有角色单图时，通过双图引用（角色图 + 转角度模板图）直接生成转角度表。

### 3. 一致性关键词是跨面板一致性的语言层面保障

多个来源独立强调了一组高度重叠的一致性关键词（来源 [01][02][05][06][07]）：
- "consistent proportions" / "exact same proportions"
- "identical styling" / "same styling"
- "exact same face" / "maintain identity"
- "do not change appearance"

这些关键词应放置在 prompt 尾部作为约束指令，形成"不变量锚点"。

### 4. 日系动漫角色设定页有明确的结构范式

来源 [06] 确立了日系动漫角色设定页的标准组件集，与碧蓝航线角色高度匹配：
- 全身转角度（正面/侧面/背面）
- 一致比例声明
- 多表情展示
- 服装分解图
- 配饰/手部/鞋子等 close-up 细节

16:9 横版是日系设定页的推荐比例（来源 [04][06]）。

### 5. 画幅与布局的共识

- 3:2 横版：适合转角度表（来源 [01]）
- 16:9 横版：适合内容丰富的完整角色设定页（来源 [04][06]）
- 3x3 网格：适合表情/姿势展示表（来源 [02]）

### 6. 与碧蓝航线角色的适配性分析

碧蓝航线角色属于日系动漫游戏风格，适配性极高：
- "professional Japanese character setting page" 关键词可直接使用（来源 [06]）
- 日系设定页的 close-up 细节组件适合展示舰装、武器等标志性元素（来源 [06]）
- 奇幻风格的种族特征锚点方法可借鉴到碧蓝航线的阵营/种族特征（来源 [04]）
- 色板（color palette）组件有助于保持角色主色调一致（来源 [04]）

## 提示词结构拆解

### 角色设定集/转角度表类 Prompt（#08, #09, #10, #43-50）

这些 prompt 共同目标是生成包含多视角、多表情、服装细节的完整角色设定集。以下基于 [01][02][03][04][05][06][07] 中描述的模板结构进行 8 维度拆解。

| 维度 | 共同模式 |
|------|----------|
| 主体清晰度 | 所有设定集 prompt 都以明确的角色身份描述开头，包含姓名、种族/阵营、年龄、体型、发色、瞳色、发型等基础属性。碧蓝航线角色需额外包含舰种、阵营、标志性舰装等游戏设定属性。角色描述在多面板间逐字复用（来源 [02][05]） |
| 构图策略 | 双层构图策略：(1) 画幅层面选择 3:2 或 16:9 横版；(2) 面板层面将画面划分为转角度区、表情区、服装细节区、色板区。转角度区通常占据最大面积，包含正面/侧面/背面三个子面板。T-pose 用于展示角色轮廓，A-pose 用于自然站姿。纯色背景（白色或暖灰 #D5D0C8）确保面板间无背景干扰（来源 [01][02][04][06][07]） |
| 风格锚点 | 设定集 prompt 使用单一明确的风格锚点：(1) 日系风格使用 "professional Japanese character setting page"（来源 [06]）；(2) 写实风格使用 "cinematic lighting"、"photorealistic"；(3) 游戏概念风格使用 "production-ready character sheet"（来源 [04]）。避免堆叠多个风格词（来源 [05][07]） |
| 光线描述 | 设定集通常采用影棚级均匀光照（studio lighting），确保各面板间光影一致。具体光线描述包括 "even studio lighting"、"clean white backdrop with soft shadows"、"no harsh shadows"（来源 [01][04]）。光线描述在设定集中是辅助角色，不作为视觉焦点 |
| 材质细节密度 | 设定集 prompt 的材质描述集中在服装和配饰：(1) 服装材质（丝绸、金属铠甲、皮革等）；(2) 配饰材质（宝石、金属、布料等）；(3) 肤质描述。日系设定页特别强调 close-up 细节面板展示材质（来源 [06]）。奇幻角色需描述魔法材质（发光纹路、能量护甲等）（来源 [04]） |
| 色彩策略 | 角色设定集通常包含色板（color palette）组件，列出角色的主色、强调色、阴影色。碧蓝航线角色需精确匹配官方配色方案。色彩描述需包含具体色名而非笼统描述（来源 [04][05]） |
| 文字规则 | 设定集 prompt 中文字内容较少，主要是面板标签（如 "Front View"、"Side View"、"Back View"、"Expression 1: Happy"）和角色名称。部分模板要求标注比例参考线。角色名在多面板中重复出现以强化身份关联（来源 [01][04]） |
| 失败规避 | (1) 使用一致性关键词约束跨面板变化（来源 [01][02][05]）；(2) 负面约束覆盖手部、面部、颈部、肩膀、手指等常见失败点（来源 [07]）；(3) 避免混合多种画风（来源 [07]）；(4) 从简单布局开始，确认一致性后再增加复杂度（来源 [02]）；(5) 角色描述逐字复用，不使用近义词（来源 [02][05]） |

### 角色设计/海报类 Prompt（#11-#41）

这批 prompt 来自 X 平台社区，覆盖各类角色设计、角色海报、游戏角色等。以下基于 [01]-[07] 的指导原则进行 8 维度归类分析。

| 维度 | 共同模式 |
|------|----------|
| 主体清晰度 | 社区 prompt 倾向于用更简洁的角色描述（1-2 句），但对关键身份特征（发色、瞳色、服装核心元素）的描述仍然精确。与设定集 prompt 相比，海报类 prompt 更注重角色的动态感和情绪表达，而非静态的外观记录 |
| 构图策略 | 海报类 prompt 使用多样化的构图：(1) 半身特写（bust shot）；(2) 全身动态姿势；(3) 斜侧角度增加动感。通常不使用网格布局，而是单面板构图。留白策略与角色设定集不同——海报追求视觉冲击力，通常减少留白 |
| 风格锚点 | 社区 prompt 的风格锚点更加多样，常见组合包括：(1) 单一风格词 + 画质修饰词（如 "anime style, masterpiece, best quality"）；(2) 参考艺术家风格。存在堆叠多个风格词的倾向，这是反面案例（来源 [01][05]） |
| 光线描述 | 海报类 prompt 的光线描述比设定集更丰富：(1) 戏剧性光线（dramatic lighting）；(2) 逆光/边缘光（rim light, backlight）；(3) 环境光（ambient light）。光线不仅是均匀照明工具，更是情绪表达手段 |
| 材质细节密度 | 海报类 prompt 的材质描述密度通常低于设定集，集中在视觉焦点区域（如服装、武器、头发）。不追求全面的材质记录，而是突出最有视觉冲击力的质感 |
| 色彩策略 | 海报类 prompt 常指定整体调性（如 "cool tone"、"warm atmosphere"）或对比策略（如 "high contrast"），而非具体色板。色彩服务于情绪和氛围 |
| 文字规则 | 海报类 prompt 基本不包含文字内容要求 |
| 失败规避 | 社区 prompt 的失败规避意识较弱，较少包含明确的负面约束。可从 [07] 的负面约束清单中补充 |

### 对话式/服装时尚类 Prompt（#42）

| 维度 | 分析 |
|------|------|
| 主体清晰度 | 服装时尚类 prompt 侧重于穿搭效果展示，角色身份描述相对次要。重点在服装的款式、材质、穿搭组合 |
| 构图策略 | 通常采用全身或 3/4 身构图，展示完整穿搭。可能包含多角度但不是标准的 turnaround 格式 |
| 风格锚点 | 时尚风格锚点（如 "streetwear"、"elegant formal"、"casual chic"），不同于角色设定的艺术风格锚点 |
| 光线描述 | 时尚摄影风格光线（柔光、环形灯效果），强调服装质感和色彩还原 |
| 材质细节密度 | 服装材质是核心描述内容，密度高 |
| 色彩策略 | 整体配色和谐度是关键 |
| 文字规则 | 可能包含品牌/设计师名称 |
| 失败规避 | 服装剪裁准确性是关键约束 |

## 提取的 pattern

### Pattern 1：身份锚点 + 一致性关键词双保障

**结构**：在 prompt 中同时使用固定角色描述词（身份锚点）和一致性关键词（语言约束），形成双重保障。

**出现频率**：所有 7 个 guide 来源均涉及（[01][02][03][04][05][06][07]）

**适用场景**：任何需要跨面板/跨生成保持角色一致性的场景。

**碧蓝航线适配**：将角色的官方设定（发色、瞳色、舰装、阵营标识）固化为身份锚点描述块，配合一致性关键词使用。

---

### Pattern 2：参考图锚定 + Edit 端点工作流

**结构**：(1) Step 1 在纯色背景上生成角色参考图；(2) Step 2 使用 edit 端点引用参考图，prompt 中明确"保留身份，只改环境/姿势"。

**出现频率**：[02][03][07]

**适用场景**：需要多次生成同一角色不同场景/姿势的高一致性场景。

**碧蓝航线适配**：先生成碧蓝航线角色的标准设定集作为参考图，再基于设定集生成各种场景（战斗、日常、节日等）。

---

### Pattern 3：Reference-Interference 双图提取

**结构**：将角色图（语义锚）和转角度模板图（结构锚）同时传入 edit 端点，使用 "extract...using...format" 句式生成。

**出现频率**：[03]

**适用场景**：已有角色立绘/单图，需要快速生成完整转角度表。

**碧蓝航线适配**：对碧蓝航线已有官方立绘，可直接提取转角度表，无需手动编写完整角色描述。

---

### Pattern 4：日系角色设定页标准组件集

**结构**：full body turnaround + consistent proportions + multiple expressions + outfit breakdown + close-up details，16:9 横版，"professional Japanese character setting page" 关键词。

**出现频率**：[04][06]

**适用场景**：日系动漫/游戏风格角色的完整设定集生成。

**碧蓝航线适配**：与碧蓝航线美术风格高度匹配。close-up 面板可展示舰装细节、武器特征等。色板组件可确保后续生成的主色调一致。

---

### Pattern 5：一致性关键词清单

**结构**：在 prompt 尾部放置一组固定的约束短语，作为"不变量指令"。

**关键词集**：
- "consistent proportions" / "exact same proportions"（来源 [01][05][06]）
- "exact same face" / "identical build" / "same skin tone"（来源 [02]）
- "identical styling"（来源 [01]）
- "do not change appearance" / "maintain identity throughout"（来源 [02]）
- "maintain exact same face and body proportions across all panels"（来源 [05]）

**出现频率**：所有 guide 来源均涉及

**适用场景**：所有多面板角色生成。

---

### Pattern 6：3x3 网格表情/姿势表

**结构**：3x3 Grid 布局，9 个格子，每个格子指定明确的表情名称和/或姿势描述。

**出现频率**：[02]

**适用场景**：角色表情集、姿势集、表情-姿势组合表。

**碧蓝航线适配**：可用于生成碧蓝航线角色的情绪展示表（如开心、愤怒、害羞、严肃、惊讶等）。

---

### Pattern 7：5 元素 Prompt 公式

**结构**：Subject + Style + Lighting + Details + Purpose 五段式组织 prompt。

**出现频率**：[05]

**适用场景**：所有 GPT Image 2 prompt 的通用组织框架。

**碧蓝航线适配**：可作为碧蓝航线角色 prompt 的标准组织方式，确保信息完整且结构清晰。

---

### Pattern 8：负面约束清单

**结构**：在 prompt 末尾或独立段落列出需要避免的常见生成问题。

**约束项**：手部自然、面部自然、手指数量正确、脖子比例正常、肩膀对称、四肢长度一致、不要混合多种画风、阴影方向一致。

**出现频率**：[07]

**适用场景**：所有角色生成，尤其是全身照和转角度表。

**碧蓝航线适配**：碧蓝航线角色通常有复杂服饰和武器，手部、颈部、肩膀等部位更容易出现生成问题，负面约束尤为重要。

## 结论

### 确认有效

1. **Step 2 基于参考图编辑场景的机制基本有效**：樱花场景氛围营造出色，角色姿态与环境融合自然。参考图能传递整体角色特征（发色、制服、配饰），但面部细节存在可感知偏差。
2. **简化布局可提高提示词遵循度**：Round 2 的最佳输出（22/30）说明减少面板数量确实让模型更容易遵循指令。

### 确认无效 / 局限性

1. **GPT Image 2 无法在单次生成中可靠产出结构化多视角设定集**：两轮验证（复杂布局和简化布局）均未达到通过阈值。多视角间面部一致性、文字标签可读性、结构化布局遵循度均不稳定。
2. **跨视角一致性是根本瓶颈**：即使在仅要求 3 个视角的情况下，模型仍无法保持跨视角的面部和体型一致性。

### 实际建议

- **推荐路径**：使用 `reference-driven-character-consistency` recipe（已 confirmed），先生成单张纯净背景参考图，再基于参考图编辑场景。该路径在 Research ① 中已验证达到 98.3% 通过率。
- **角色设定集的替代方案**：如需多视角参考，可通过多次单视角生成 + 后期拼接实现，而非依赖单次生成。
- **本 recipe 保持 draft 状态**：核心目标（单次生成结构化设定集）未验证通过，但 Step 2 编辑模式和布局简化策略的研究结论仍有参考价值。
