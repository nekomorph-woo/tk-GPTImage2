## 指导要点

- **两步角色一致性工作流**：Step 1 创建参考图（纯暖灰影棚背景）→ Step 2 编辑生成（引用参考图，只改环境/姿势，保留身份/服装）
- **参考图创建要求**：干净的暖灰色背景（`#D5D0C8` 或类似），角色正面全身照，自然站姿
- **Edit Prompt 核心句式**："使用参考角色，保留 [角色名] 的脸、发型、服装...保持身份和服装一致；只改变环境和姿势"
- **负面约束（来自 NEGATIVE_CONSTRAINTS.md）**：手部自然、面部自然、手指数量正确、脖子比例正常、肩膀对称、四肢长度一致、不要混合多种画风、阴影方向一致
- **十维度提示词框架**：`[格式] + [主体] + [构图] + [风格锚点] + [光线] + [材质] + [色彩] + [文字] + [约束] + [迭代指令]`，其中主体维度需包含完整的角色身份描述
- **角色一致性原则**："需要角色一致性或产品一致性时，优先使用参考图编辑"

## 原文摘要

项目内部魔导书第 5.5 节详细描述了使用 GPT Image 2 实现角色一致性的工作流程。第一步在干净的暖灰背景上生成角色参考图，第二步使用 edit 端点引用该参考图生成不同场景/姿势的图片，prompt 中明确要求"保留脸、发型、服装，只改环境和姿势"。

配合项目的 NEGATIVE_CONSTRAINTS.md 负面约束（手部、面部、解剖、风格控制、光影色彩等类别），以及 PROMPT_FRAMEWORK.md 的十维度提示词框架，形成了一套完整的角色一致性生成方法论。
