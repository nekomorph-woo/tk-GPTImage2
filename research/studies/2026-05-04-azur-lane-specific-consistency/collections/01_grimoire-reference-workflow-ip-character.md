## 指导要点

- **两步工作流是 IP 角色一致性的核心范式**：先做参考图（generate），再做场景编辑（edit），比纯文本反复生成更可靠。
- **参考图 prompt 的关键要求**：
  - 明确角色名字和身份（如"Enterprise, Azur Lane"）
  - 纯净背景（暖灰色摄影棚），不掺杂场景元素
  - 标准站姿（三分之四站姿），脸和手清晰可见
  - 完整的服装和配饰描述，包含舰装等复杂元素
  - 柔和棚拍光，不使用戏剧化光线
  - 声明"不要额外配饰，不要文字"
- **场景编辑 prompt 的关键要求**：
  - 开头声明"使用参考角色"
  - 显式列出需要保留的具体特征（脸、发型、服装、舰装、配饰）
  - 明确"保持身份和服装一致"
  - 限定"只改变环境和姿势"
- **关键心法**："要稳定角色或产品，不要只靠文本反复生成；先做参考图，再用编辑保持身份和结构。"
- **迭代原则**：一次只改一个变量（光线、构图、色彩、材质、文字），不要一起大改。
- **"一个提示词只放一个强风格锚点"**：避免混合多种风格导致输出不稳定。

## 原文摘要

项目内部魔导书在"5.5 角色一致性"章节确立了参考图工作流为 IP 角色一致性的核心范式。以角色"Lin"为例，第一步用 generate 创建纯净背景下的全身参考图，第二步用 edit 将角色放入新场景。魔导书强调"先做参考图，再用编辑保持身份和结构"比纯文本反复生成更可靠。该工作流被后续 Enterprise 角色测试直接验证，证明对碧蓝航线等复杂配饰角色的适用性。
