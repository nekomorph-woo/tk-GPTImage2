# 02 多参考图 API 能力——IP 角色一致性强化

- **来源**: [Fal.ai GPT Image 2 Prompting Guide](https://fal.ai/learn/tools/prompting-gpt-image-2) + [NoteGPT GPT Image 2 Guide](https://notegpt.io/blog/gpt-image-2-prompt-guide-use-cases)
- **来源类型**: 博客指南
- **可信度**: 二手来源
- **素材类型**: guide
- **采集时间**: 2026-05-04

## 指导要点

- **gpt-image-2 支持多达 16 张参考图输入**：API 的 edit 端点接受最多 16 张参考图片（file ID 或 URL），远超许多用户的认知。
- **多参考图对 IP 角色的应用模式**：
  - 上传同一 IP 角色的多角度官方立绘（正面、侧面、背面），让模型综合理解角色外貌
  - 上传角色设定页 + 目标场景参考，实现 IP 角色在新场景中的渲染
  - 上传多张角色图作为风格参考，保持画风一致
- **角色设计表作为参考图**：先生成完整的角色设计表（character sheet），再将该表作为后续所有生成的唯一参考图，这是社区广泛采用的工作流。
- **Fresh Chat 策略**：每次开始新对话配合参考图上传，可以有效避免上下文漂移，对需要多次生成的 IP 角色尤为重要。
- **编辑端点 vs 生成端点**：编辑端点（edit）比生成端点（generate）更适合 IP 角色一致性场景，因为编辑端点会显式参考输入图片。

## 原文摘要

Fal.ai 的技术文档确认 gpt-image-2 的 edit 端点支持多达 16 张参考图输入，NoteGPT 验证了 ChatGPT 网页界面支持 8 张图片上传。这一多参考图能力为 IP 角色一致性打开了多种应用模式：多角度立绘综合、设定页+场景组合、多角色风格对齐等。社区最佳实践包括先生成角色设计表再将其作为后续所有编辑的唯一参考、使用 Fresh Chat 策略避免长上下文漂移、以及链式编辑工作流。
