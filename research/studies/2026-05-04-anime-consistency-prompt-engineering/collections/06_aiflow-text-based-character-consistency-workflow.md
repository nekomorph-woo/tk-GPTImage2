# 06 AI-FLOW 指南：纯文本角色定义与多场景一致性工作流

- **来源**: [Creating Consistent Characters for Storytelling with GPT Image and AI-FLOW](https://ai-flow.net/blog/consistent-character-generation-workflow-gpt-image/)
- **来源类型**: 博客指南
- **可信度**: 二手来源
- **素材类型**: guide
- **采集时间**: 2026-05-04

## 指导要点

- **文本角色定义的核心要素**：
  - Physical Appearance：年龄、体型、服装、发型、面部特征
  - Unique Traits：特殊标记、配饰、特征色系
- **精确的 Prompt 是一致性的基础**：包含角色特质、动作和环境的精确描述
- **视觉标记物策略**：在每次 prompt 中一致地包含独特的视觉标记（如独特的配饰、色系、服装元素），这些标记在跨场景时提供识别锚点
- **可扩展的工作流**：
  - 快速生成角色的多个变体/场景
  - 确保跨迭代叙事中视觉表示的一致性
  - 执行并行工作流以提高大型项目的生产力
- **适用场景**：图形小说（保持角色跨章节可识别）、教育材料、品牌和营销（一致的品牌吉祥物）、动画内容

## 原文摘要

AI-FLOW 的指南强调了从清晰的文本角色定义开始的重要性。该工作流的核心是：首先用纯文本完整定义角色的物理外观和独特特征，然后通过精确的 prompt 在不同场景中复现角色。关键策略是使用一致的"视觉标记"（distinctive accessories, color schemes, clothing items）来提供跨场景的识别锚点，而不依赖参考图。该工作流支持并行执行，适合需要大量一致角色图像的项目。
