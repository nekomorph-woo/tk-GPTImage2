# GPT Image2 提示词框架

当你追求质量、可复现和可迭代时，优先使用这个结构。

```text
[目的]
这张图要承担什么任务：海报、编辑图、产品渲染、角色设定、UI 资产、封面、场景图。

[主体]
准确描述主要对象、可见特征、姿态/状态，以及必须保持一致的内容。

[构图]
镜头角度、裁切、主体大小、前景/中景/背景、留白、视觉层级。

[风格锚点]
一个具体视觉语言：电影剧照、高级产品摄影、杂志编辑视觉、赛璐璐 3D、Risograph 印刷海报。

[光线]
光源方向、软硬、色温、对比度、高光表现、阴影表现。

[材质细节]
表面、纹理、镜头痕迹、印刷工艺、织物、金属、玻璃、皮肤、纸张、尘埃、颗粒。

[色彩策略]
2-4 个主色、一个强调色、饱和度规则、背景和主体的色彩关系。

[文字规则]
是否需要精确文字、语言、位置、字体气质、可读性约束。

[约束]
需要避免什么：多余手指、变形文字、重复 Logo、背景杂乱、角色身份不一致。

[迭代指令]
用于编辑：保留构图/身份，只修改指定视觉层。
```

## 稳定提示词公式

```text
创建一张[格式]，主体是[主体]。
构图：[镜头/裁切/布局]。
视觉语言：[风格锚点]。
光线：[光线]。
材质与细节：[材质]。
色彩：[调色板]。
文字：[精确文字与位置，或“无文字”]。
质量约束：[具体失败规避]。
```

## 中英混合原则

创意构思可以先用中文完成，最终生成提示词可以使用精确英文，也可以使用结构清晰的中文。专有名词、精确标题和必须出现的文字保持原样。

## 稳定性规则

- 测试时一次只改一个变量。
- 先固定主体和构图，再探索风格。
- 需要角色一致性或产品一致性时，优先使用参考图编辑。
- 不要堆叠太多风格名；使用一个强风格锚点，加上具体视觉细节。
- 精确文字放在引号中，并尽量短。
- 每个有价值结果都记录到 `studies/`，写清 prompt 和它为什么有效。
