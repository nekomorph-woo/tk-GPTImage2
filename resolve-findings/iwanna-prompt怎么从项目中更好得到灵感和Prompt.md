# iwanna-prompt 灵感发现与人类概念层

- **状态**: ✅ 已解决
- **发现日期**: 2026-05-04
- **决策**: 在步骤 1 之前新增独立步骤 0「灵感发现 + 概念塑造」

## 发现

### 问题 1：灵感发现断裂

用户输入模糊想法时，步骤 2 的素材检索是被动搜索（需要用户给出具体关键词才 Grep），缺少主动展示项目资源的能力。用户经常处于"我不知道项目里有什么可以参考"的状态。

### 问题 2：缺少人类概念层

当前技能让用户直接面对流派、构图、材质等专业维度（步骤 1 的 AskUserQuestion 要求选择风格/画幅/文字），对不了解生图 prompt 的初学者（如 20% 掌握程度的小白）不友好。

**核心洞察**：用户更擅长用比喻、白描、临场描绘来表达"想要什么感觉"，而不是选择"写实摄影/电影剧照/产品海报"等流派。大模型可以填补"人类感觉"和"结构化 prompt"之间的翻译层。

**深层洞察**：这一层本质上是生图的**源头背景**——需求怎么来的、沟通过程中产生了什么想法、生活中遇到了什么场景、工作任务需要什么图。生图从不凭空而来，即使是"突然想试试"，背后也有一个源头（听说某个模型厉害、想把以前的想法可视化、工作上需要一张图）。对话应该从"你为什么想生这张图"开始，而不是"你想要什么风格/构图/材质"。

## 决策

### 新增步骤 0：灵感发现 + 概念塑造（在步骤 1 之前）

分两个子阶段执行：

**子阶段 A — 灵感发现**（展示项目资源）：

展示以下内容帮助用户找到方向：
- 覆盖领域分布图（按用途类别分组，展示项目有哪些品类）
- 可用模板推荐（recipes/confirmed + collected/ 中 template 标签）
- 模糊关键词联想（根据用户输入推荐 3 条素材）
- 热门流派和标签（标签 TOP 10 + 流派分布）

**子阶段 B — 概念塑造**（渐进式人机对话）：

1. **先聊源头**：从"你为什么想生这张图"开始聊——是工作任务需要？看到某个图想复刻？脑海里一直有个画面想可视化？生活中的某个瞬间想留住？聊出需求的源头背景
2. **再描述场景**：用自然语言向用户描述 2-3 个场景（比喻、白描、临场描绘），让用户选择或调整。输出一段人类可读的"概念描述"（不包含任何 prompt 技术术语）
3. **最后看技术解读**：将概念翻译成流派/构图/材质/色彩等技术维度（左右对照格式），让用户确认或微调

确认后才进入后续步骤（流派匹配 → 构造 prompt）。

## 终极定位

一句生活随笔、一条工作备注、悲伤时的小作文、高兴时的 YOHO 大叫——都是生图的来源。去描述现实、去具象化情感、去抽象神秘的概念，就是这个中间层的职责：**向上与人类知心沟通，向下对接技术层，为最终的 Prompt 负责**。

这不是一个"帮用户填表"的工具，而是一个"和用户一起发现想要什么"的伙伴。

## 对话人格

步骤 0B 赋予执行人格：**苏格拉底式提问** + **温暖的、好奇的、专注于画面感的朋友语气**。

三个语气锚点：
- **温暖但克制**：认真倾听、复述确认，不过度共情
- **好奇但不审问**：提问出于对画面本身的好奇，不像填表
- **接住情绪但不沉溺**：从情绪中提取视觉线索，情绪是素材不是话题

终极效果：当画面通过聊天在用户脑子里清晰展现时，用户自己就有了生成目标——而不是最终生图时因为技术精妙而忽略了自己真正想要什么。

## 关键原则

- 对话起点是"为什么想生图"（源头），不是"想要什么风格"（结果）
- 概念描述阶段禁止出现 prompt 技术术语（流派名、构图术语、材质名词）
- 用户确认的"概念描述"作为步骤 2-5 的输入，替代当前的 AskUserQuestion 直接选风格
- 概念描述格式参考：一段 3-5 句的自然语言，像产品经理向设计师展示需求概念

## 影响范围

- `SKILL.md`：新增步骤 0 定义，步骤 1 简化为读取概念描述（不再用 AskUserQuestion）
- `references/`：可能需要新增概念塑造的输出格式参考
- 步骤 2-5 的输入从"用户原始输入 + 步骤 1 回答"变为"概念描述 + 技术解读"
